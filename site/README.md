# Система мониторинга

Разработка системы мониторинга, анализа информации и создание платформенного решения на основе ИИ для прогнозирования социально-экономических показателей предприятий и региональной власти

## Настройка рабочего окружения

**Приступая к работе:**

**Установка зависимостей и запуск проекта:**

Требуется **Node.js: 22.x (LTS)**

```bash
# Установка зависимостей
$ npm install

# Запуск в режиме разработки на localhost:5173
$ npm run dev
```

## Текущие разделы

- `/` — главная страница с метриками из БД, прогнозными ориентирами и быстрыми действиями
- `/dashboards` — аналитический дашборд по target-показателям: выбор модели/сценария, график факта и прогноза, прогнозные карточки, SHAP-анализ с режимами `Топ факторов` и `Все факторы`
- `/forecasts` — сводка прогнозов с автоматической загрузкой таблицы, KPI-карточками модели, сценарными бейджами, sticky-колонками и экспортом в Excel-совместимый `.xls`
- `/indicators` — проверка фактических данных: таблица target/features, карточки полноты данных, график выбранного фактора, фильтр по сфере и статистика ряда
- `/data-upload` — загрузка данных за новый год: скачивание Excel-шаблона, загрузка заполненного файла, валидация, сохранение и удаление значений за год
- `/scenarios` — what-if сценарное моделирование на основе SHAP-факторов: выбор модели/сценария, ползунки факторов, KPI отклонения, график и таблица пользовательского сценария

Фронтенд получает данные через FastAPI-прокси Vite (`/api` → `http://localhost:8000`).

## Работа с загрузкой данных

Панель `/data-upload` использует безопасный Excel-сценарий:

1. Пользователь выбирает target-показатель.
2. Скачивает `.xlsx`-шаблон для следующего года.
3. Заполняет только колонку `Значение`.
4. Загружает заполненный шаблон обратно.
5. Интерфейс показывает найденные значения и ошибки.
6. После проверки пользователь сохраняет данные в БД.

Backend endpoints:

- `GET /api/dataset/template` — генерирует Excel-шаблон.
- `POST /api/dataset/template/parse` — читает заполненный шаблон.
- `POST /api/dataset/year` — сохраняет значения.
- `DELETE /api/dataset/year` — удаляет загруженные значения за год для выбранного target.
